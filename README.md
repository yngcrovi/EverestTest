# Документация

## Запуск проекта

### ВАЖНО!!!
У вас может отличаться версия docker compose, есть представленная команда не работает, то попробуйте так:
```console
sudo docker-compose ...
```

### Сбилдить образ проекта
```console
sudo docker compose --env-file ./src/.env build
```

### Создать копию файла .env из .env.example(обязательно внутри папки ./src!!!)
```console
cd src/
cp .env.example .env
```
Заполнить его данными, которые пришлю

### В корне проекта запустить контейнеры, указав относительный путь к .env
```console
sudo docker compose --env-file ./src/.env up
```
Подождать выполнение программы, вся инофрмация о выполнения будет в логах контейнера парсера, либо в файле /log/log.txt
Также можно запустить приложение в фоновом режиме с флагом -d

### Посмотреть результат
После выполения програмымы(может занимать от 10 до 15 минут), можно посмотреть результат внутри контйенера postgres(либо в новом окне, либо запустить контейнеры в режиме demon):
```console
sudo docker exec -it everest_db bash
psql -U postgres -d everest
```
Затем выполнить эти sql команды:
```sql
SELECT * FROM people LIMIT 5;
SELECT * FROM bankruptcy_info LIMIT 5;
SELECT * FROM bankruptcy_docs_data LIMIT 5;
```

### Завершение работы
После того, как всё успешно спарсится, сайт будет просто весеть в ожидании действий, можно выключать программу либо очистив контейнеры(смотреть следующий пункт), либо нажав ctrl+C

### Очистить базу и выключиь контейнеры
```console
sudo docker compose down -v
```

## Дополнительная информация

### Обоснование выбора парсера
1. Работа с динамическим контентом
- Сайты fedresurs.ru и kad.arbitr.ru активно используют JavaScript

- Playwright запускает реальный браузер (Chromium) и может ждать загрузки динамических элементов

- В отличие от requests/BeautifulSoup, видит контент, который подгружается после выполнения скриптов

2. Автоматизация действий
- Возможность эмулировать клики, ввод текста, навигацию

- Поддержка ожидания появления элементов (wait_for_selector)

3. Асинхронность
- Нативная поддержка async/await позволяет эффективно распараллеливать запросы

- Возможность открывать несколько вкладок в одном браузере для ускорения

4. Кроссплатформенность
- Работает в Docker, Linux, Windows, macOS

- Поддержка headless-режима для серверов без графического интерфейса

5. Надежность
- Встроенные механизмы ожидания и повторных попыток

6. Альтернативы не подошли:
- requests/BeautifulSoup - не видят JavaScript-контент

- Selenium - медленнее, сложнее в настройке, больше потребляет ресурсов

- Scrapy - избыточен для данного проекта, сложнее интегрировать с асинхронностью

### Работа программы
- Список ИНН я беру с самого сайта 'https://fedresurs.ru', из результатов пустого поиска я беру ИНН пользователей, данные загружаются по 15 человек, параметр, отвечающий за количество таких подгрузок лежит в .env файле и называется QUANTITY_INN(его можно менять). По умолчанию беру 105 ИНН, сохраняю их в таблицу people и дальше уже обрабатываю их
- Вся структура БД PostgreSQL лежит в файле ./sql/init.sql, который расположен относительно корня проекта
- Все cookie взял с соотвествующих сайтов
